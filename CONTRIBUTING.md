# Contributing to TCD Origin

Thank you for your interest in contributing to TCD Origin - 跨文明古文字拓扑破译引擎!

## 🤝 How to Contribute

### Reporting Issues

- **Bug Reports**: Use GitHub Issues with the `bug` label
- **Feature Requests**: Use GitHub Issues with the `enhancement` label
- **Questions**: Start a Discussion

### Pull Request Process

1. **Fork the Repository**: Create your own copy of the project
2. **Create a Branch**: Work on a specific feature or fix
   ```bash
   git checkout -b feature/amazing-feature
   # or
   git checkout -b fix/annoying-bug
   ```
3. **Make Changes**: Follow our coding standards
4. **Test Your Changes**: Ensure all tests pass
5. **Commit**: Write clear, descriptive commit messages
6. **Push**: Push your changes to your fork
7. **Submit PR**: Open a Pull Request with a clear description

## 📋 Coding Standards

### Python Code

- Follow PEP 8 style guidelines
- Add type hints for function parameters and return values
- Write docstrings for all public functions and classes
- Keep functions focused and small (ideally < 50 lines)

### Documentation

- Update docstrings when changing code
- Add examples for new features
- Update README.md if needed
- Translate to English if adding new content

### Testing

- Write unit tests for new features
- Ensure existing tests pass
- Aim for > 80% code coverage
