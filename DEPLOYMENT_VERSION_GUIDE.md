## 🌟 方案一：一键部署（强烈推荐）

### ✅ 优点

- **最简单** - 只需1个命令
- **自动化** - 自动配置所有环境
- **零门槛** - 无需任何配置经验
- **快速** - 5-10分钟完成部署
- **完整** - 包含所有功能组件

### 📦 包含组件

| 组件 | 说明 |
|------|------|
| ✅ **Web应用** | FastAPI + 古文字破译Agent |
| ✅ **数据库** | PostgreSQL 16 |
| ✅ **缓存** | Redis 7 |
| ✅ **反向代理** | Nginx |
| ✅ **健康检查** | 自动监控服务状态 |
| ✅ **数据持久化** | 自动数据卷管理 |
| ✅ **日志系统** | 完整日志记录 |

### 🚀 快速开始

```bash
# 1. 进入项目目录
cd /path/to/ancient-script

# 2. 运行一键部署脚本
./deploy.sh

# 3. 输入 Moonshot AI API Key
#    访问：https://platform.moonshot.cn/console/api-keys

# 完成！
```

### 🌐 访问地址

部署完成后，访问：

- 📡 **API 文档**: http://localhost:8000/docs
- 🌐 **前端界面**: http://localhost:8000/static/index.html
- ❤️ **健康检查**: http://localhost:8000/health

### ⚙️ 常用命令

```bash
# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 重启服务
docker-compose restart

# 停止服务
docker-compose down

# 完全删除（包括数据）
docker-compose down -v
```

### 📝 环境变量

一键部署会自动配置以下环境变量：

| 变量名 | 说明 | 自动生成 |
|--------|------|----------|
| `COZE_WORKLOAD_IDENTITY_API_KEY` | Moonshot AI API Key | ❌ 需输入 |
| `DB_PASSWORD` | 数据库密码 | ✅ 自动生成 |
| `REDIS_PASSWORD` | Redis密码 | ✅ 自动生成 |
| `SECRET_KEY` | 应用密钥 | ✅ 自动生成 |
| `JWT_SECRET` | JWT密钥 | ✅ 自动生成 |

### 📄 配置文件

一键部署会创建以下文件：

| 文件 | 说明 |
|------|------|
| `.env` | 环境变量配置 |
| `docker-compose.yml` | Docker Compose配置 |
| `docker-compose.override.yml` | 覆盖配置（可选） |

## 🔧 方案二：Docker Compose（高级用户）

### 适用场景

- ✅ 需要自定义配置
- ✅ 需要多环境部署
- ✅ 需要深入控制

### 📝 步骤

```bash
# 1. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，填入API Key和密码

# 2. 启动服务
docker-compose up -d

# 3. 查看日志
docker-compose logs -f
```

### 📦 自定义选项

- 自定义端口
- 自定义镜像
- 自定义卷挂载
- 自定义网络配置

## 🚨 注意事项

### 一键部署

- ⚠️ 需要安装Docker和Docker Compose
- ⚠️ 需要Moonshot AI API Key
- ⚠️ 需要至少2GB内存
- ⚠️ 需要至少10GB磁盘空间

### Docker Compose

- ⚠️ 需要手动配置环境变量
- ⚠️ 需要手动管理密码
- ⚠️ 需要手动初始化数据库

### 云部署

- ⚠️ 需要购买云服务器
- ⚠️ 需要配置域名和SSL
- ⚠️ 需要配置防火墙
- ⚠️ 需要监控和维护

## 📞 遇到问题？

### 常见问题

1. **Docker未安装**
   - 解决：参考 [Docker安装指南](https://docs.docker.com/get-docker/)

2. **API Key无效**
   - 解决：访问 https://platform.moonshot.cn/console/api-keys 获取

3. **端口被占用**
   - 解决：修改 `docker-compose.yml` 中的端口映射

4. **内存不足**
   - 解决：增加服务器内存或减少缓存大小
